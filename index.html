Theory & Fundamentals

Understanding the mathematical foundations of error detection and correction

In digital communication, data is transmitted as binary digits (0s and 1s). External factors like noise can corrupt some bits. Error detection and correction codes help identify and fix these errors, ensuring reliable data transmission.

Hamming Code Lesson

What is a Hamming Code?

Hamming code is a linear code that is useful for error detection up to two immediate bit errors. It is capable of correcting single-bit errors.

History of Hamming Code

  • Hamming code is a technique built by R.W. Hamming to detect errors.
  • It applies to data units of any length and uses the relationship between data and redundancy bits.
  • He worked on the problem of error-correction methods and developed powerful algorithms called Hamming codes.
  • In 1950, he published the Hamming Code, which is widely used today in applications like ECC memory.

Applications of Hamming Code

  • Satellites
  • Computer Memory
  • Modems
  • PlasmaCAM
  • Open Connectors

Process of Encoding a Message Using Hamming Code

The process used by the sender to encode the message includes the following steps:

  • Calculation of total number of redundant bits
  • Checking the position of the redundant bits
  • Calculating the values of the redundant bits

Step 1: Total Number of Redundant Bits

Let the message contain:

  • n ‚Äì number of data bits
  • p ‚Äì number of redundant bits added so that it can indicate at least (n + p + 1) different states.

(n + p) depicts the location of an error in each of (n + p) bit positions, and one extra state means no error. Since p bits indicate 2p states, 2p must be at least equal to (n + p + 1).

Step 2: Placing the Redundant Bits

The redundant bits should be placed in bit positions of powers of 2.

  • For example: 1, 2, 4, 8, 16, etc.
  • They are referred to as p1 (position 1), p2 (position 2), p3 (position 4), etc.

Step 3: Calculating Values of Redundant Bits

The redundant bits should be parity bits that make the number of 1s either even or odd.

Types of Parity:

  • Even Parity ‚Äì total number of 1s in the message is made even.
  • Odd Parity ‚Äì total number of 1s in the message is made odd.

Calculating Parity Bits (p1, p2, p3, ...)

All redundant bits (p1, p2, p3, ...) are calculated as parity bits. Each parity bit covers specific positions based on binary representation.

  • P1 ‚Äì Covers all bit positions whose binary representation includes 1 in the 1st position (excluding position of p1). Example: 3, 5, 7, 9...
  • P2 ‚Äì Covers positions that include 1 in the 2nd bit from right (excluding position of p2). Example: 3, 6, 7, 10, 11...
  • P3 ‚Äì Covers positions that include 1 in the 3rd bit from right (excluding position of p3). Example: 5‚Äì7, 12‚Äì15...

Process of Decrypting a Message in Hamming Code

Receiver gets incoming messages which require recalculations to find and correct errors. The process includes:

  • Counting the number of redundant bits
  • Correctly positioning all redundant bits
  • Performing parity check

Step 1: Counting the Number of Redundant Bits

You can use the same formula for encoding:

2p ‚â• n + p + 1

Here, n = number of data bits and p = number of redundant bits.

Step 2: Correctly Positioning Redundant Bits

Each redundant bit is located at bit positions that are powers of 2 (1, 2, 4, 8, etc.).

Step 3: Parity Check

Parity bits are recalculated based on data bits and redundant bits:

  • p1 = parity(1, 3, 5, 7, 9, 11‚Ä¶)
  • p2 = parity(2-3, 6-7, 10-11‚Ä¶)
  • p4 = parity(4-7, 12-15, 20-23‚Ä¶)
  • p8 = parity(8-15, 24-31‚Ä¶)

Repetition Code Lesson

What is a Repetition Code?

Repetition codes are one of the simplest types of error detection and correction codes used in digital communication systems.

Definition:

A repetition code works by repeating each bit of data multiple times to help detect and possibly correct errors that occur during transmission.

It is one of the most basic error-correcting codes used to increase reliability in communication systems.

Concept of Repetition Code

Repetition coding ensures data accuracy by transmitting each bit several times. If an error occurs during transmission, the receiver uses majority voting to determine the original bit value.

Repetition Code Formula

The formula to determine how many times each bit should be repeated is:

Formula: k = 2r + 1

Where r is the number of redundant bits added for error correction.

Example 1: Encoding Process

Suppose the source message is 101. If r = 2, what will be the transmitted message using repetition coding?

Solution:

k = 2r + 1

k = 2(2) + 1 = 5

Step 1: Repeat each bit 5 times:

  • 1 ‚Üí 11111
  • 0 ‚Üí 00000
  • 1 ‚Üí 11111

Step 2: Combine all groups:

Resulting Message: 111110000011111

Example 2: Transmission with Error

The encoded message 11111 00000 11111 was transmitted, but due to noise, it was received as 111100000011101. If r = 2, find the original source message.

Step 1: Group bits into sets of 5 (since r = 2 ‚Üí k = 5):

Groups: (11110) (00000) (11101)

Step 2: Apply majority voting for each group:

  • 11110 ‚Üí 1's = 4 ‚Üí 1
  • 00000 ‚Üí 0's = 5 ‚Üí 0
  • 11101 ‚Üí 1's = 4 ‚Üí 1

Step 3: Decoded message = 101
